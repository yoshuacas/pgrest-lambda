Replace AWS Cognito with a GoTrue-native auth provider as the default.
Store users and refresh tokens directly in PostgreSQL (DSQL-compatible),
eliminating the external Cognito dependency. Cognito stays available as
an optional provider (`AUTH_PROVIDER=cognito`).

## Context

pgrest-lambda delegates all user management to AWS Cognito today. This
creates a hard dependency on an AWS managed service. The goal is a self-
contained auth layer: users live in the same database the REST engine
already connects to, using an `auth` schema so auth tables stay out of
REST introspection (which targets `public` only).

The implementation must remain wire-compatible with `@supabase/supabase-js`.

Reference: The original auth-layer prompt (docs/design/prompts/auth-layer.md)
built the Cognito-backed system. This prompt replaces Cognito as the default
while preserving every existing contract (endpoints, JWT claims, authorizer
context, response shapes).

## Design decisions (already resolved)

1. **DB access** — GoTrue provider shares `ctx.db` (the existing pool).
   Passed via `createProvider(config, db)`.
2. **Refresh tokens** — Keep the JWT wrapper with `prt` claim holding an
   opaque DB token. Handler refresh flow (`handler.mjs:176`, `claims.prt`)
   needs zero changes.
3. **Schema init** — Auto-create with `CREATE TABLE IF NOT EXISTS` on first
   auth request. Each statement runs individually (DSQL single-statement
   constraint).
4. **Schema namespace** — `auth` schema. Invisible to REST introspection.
5. **Email confirmation** — Auto-confirm (`email_confirmed_at = now()`).
   Matches current Cognito presignup.mjs behavior.
6. **bcrypt** — `bcryptjs` (pure JS). Lambda-safe, no native compilation.

## Database schema (DSQL-compatible)

```sql
CREATE SCHEMA IF NOT EXISTS auth;

CREATE TABLE IF NOT EXISTS auth.users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT NOT NULL,
    encrypted_password TEXT NOT NULL,
    email_confirmed_at TIMESTAMPTZ DEFAULT now(),
    role TEXT NOT NULL DEFAULT 'authenticated',
    aud TEXT NOT NULL DEFAULT 'authenticated',
    app_metadata JSONB NOT NULL DEFAULT '{"provider":"email","providers":["email"]}'::jsonb,
    user_metadata JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX IF NOT EXISTS auth_users_email_idx ON auth.users (email);

CREATE TABLE IF NOT EXISTS auth.refresh_tokens (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY (CACHE 1) PRIMARY KEY,
    token TEXT NOT NULL,
    user_id UUID NOT NULL,
    parent TEXT,
    revoked BOOLEAN NOT NULL DEFAULT false,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX IF NOT EXISTS auth_refresh_tokens_token_idx ON auth.refresh_tokens (token);
CREATE INDEX IF NOT EXISTS auth_refresh_tokens_user_id_idx ON auth.refresh_tokens (user_id);
```

No foreign keys, no SERIAL, no multi-statement, no enums — all DSQL constraints.

## Requirements

### 1. New file: `src/auth/schema.mjs`
- Export `AUTH_SCHEMA_SQL` — array of CREATE statements (one per element)
- Export `ensureAuthSchema(pool)` — runs each statement sequentially,
  guarded by module-level `initialized` flag (once per cold start)
- Export `_resetInitialized()` for testing

### 2. New file: `src/auth/providers/gotrue.mjs`

Implements the AuthProvider interface from `src/auth/providers/interface.mjs`.

Constructor: `createGoTrueProvider(config, db)` where `db` has `getPool()`.
Returns `{ provider, _setClient: null }` to match the existing factory shape.

**signUp(email, password):**
- `ensureAuthSchema(pool)`
- Validate password: 8+ chars, uppercase, lowercase, number.
  Throw `{ code: 'weak_password', reasons: [...] }` on failure.
- Hash with `bcryptjs.hash(password, 10)`
- INSERT into auth.users RETURNING id, email, app_metadata, user_metadata, created_at
- Catch PG error code `23505` → throw `{ code: 'user_already_exists' }`

**signIn(email, password):**
- `ensureAuthSchema(pool)`
- SELECT user by email
- If no row → dummy bcrypt compare (timing attack prevention) → throw `{ code: 'invalid_grant' }`
- `bcryptjs.compare(password, encrypted_password)`
- On mismatch → throw `{ code: 'invalid_grant' }`
- Generate opaque refresh token: `crypto.randomBytes(16).toString('base64url')`
- INSERT into auth.refresh_tokens
- Return `{ user, providerTokens: { refreshToken: opaqueToken } }`

**refreshToken(opaqueToken):**
- SELECT refresh token row by token value
- If not found → throw `{ code: 'invalid_grant' }`
- If revoked → revoke ALL tokens for this user (family revocation) → throw `{ code: 'invalid_grant' }`
- Generate new opaque token
- INSERT new token with `parent = oldToken`
- Revoke old token (UPDATE SET revoked = true)
- Fetch user by user_id
- Return `{ user, providerTokens: { refreshToken: newToken } }`

**getUser(userId):**
- SELECT user by id

**signOut():**
- No-op (same as Cognito)
